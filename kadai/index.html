コマンド 説明 使用例
add 全てのファイルをコミット対象として追加する git add .
add 任意のファイルをコミット対象として追加する git add ${コミット対象として追加するファイル名}
apply 過去の編集内容をパッチとして当てることで復活させる方法。
1.対象のコミットのコミットIDを確認（コミットログの文字列で検索）
2.特定のコミットからブランチを作成
3.ひとつ前との差分を取り、パッチを作成
4.差分をパッチとして適用 		git log --grep ${検索文字列}
git checkout -b ${ブランチ名} ${作成元コミットID}
git diff HEAD^ > ${パッチファイル名}
git apply -p1 ${パッチファイル名}
branch ブランチの一覧を表示 git branch
branch 現在のブランチにマージ済みのブランチを調べる git branch --merged
branch 現在のブランチに未マージのブランチを調べる git branch --no-merged
branch ブランチを削除 git branch -d ${ブランチ名}
branch ブランチを削除(未マージのブランチを強制削除) git branch -D ${ブランチ名}
branch 各ブランチにおける直近のコミットを調べる git branch -v
branch ブランチを新規作成 git branch ${ブランチ名}
checkout ブランチを作成して、そのブランチに切り替える git checkout -b {ブランチ名}
checkout ブランチを切り替える git checkout ${ブランチ名}
clone リポジトリの複製（クローン）を作成する git clone ${複製するリポジトリのURL} ${opt-ローカルのディレクトリ名}
commit コミットを実行する git commit
commit 直前のコミットを取り消してコミットする git commit --amend
commit オプションで変更されたファイルを自動検出してコミットする。 git commit -a
commit コミットメッセージを再利用する git commit -c some-commit
commit オプションでコメントを引数として指定できる。 git commit -m
commit 変更点を表示しながらコミットメッセージを編集してコミットする git commit -v
commit 指定したファイルのコミットを実行する git commit ${コミットするファイル名}
diff 作業ディレクトリと特定のコミットの変更点をdiffで表示する git diff ${コミットID}
diff 作業ディレクトリとステージングの変更点をdiffで表示する git diff ${ファイル名}
fetch リモートリポジトリから最新のコミットを取得する(pullと異なりマージは行わない) git fetch ${リポジトリ名} 又は git fetch ${URL}
gitk コミットグラフ・diff等のユーティリティ表示 gitk
gitk --all すべてのブランチを表示 gitk --all
gitk --max-count=${number} ${number}で指定された数のコミットを表示 gitk --max-count=${number}
gitk --select-commit=${ref} ${ref}で指定されたコミットをカレント位置にして表示 gitk --select-commit=${ref}
gitk --since=${date} ${date}で指定された日付以降のコミットを表示 gitk --since=${date}
gitk --until=${date} ${date}で指定された日付以前のコミットを表示 gitk --until=${date}
gitk -n ${number} ${number}で指定された数のコミットを表示 gitk -n ${number}
help コマンドの一覧を表示 git help
help -a 利用可能なコマンドとパラメータの一覧を表示 git help -a
help -i ${command} ${command}で指定されたコマンドの詳細ヘルプをinfo形式で表示 help -i ${command}
help -m ${command} ${command}で指定されたコマンドの詳細ヘルプをman形式で表示 help -m ${command}
help -w ${command} ${command}で指定されたコマンドの詳細ヘルプをhtml形式で表示 htlp -w ${command}
help ${command} ${command}で指定されたコマンドの詳細ヘルプを表示 git help ${command}
log コミットログを表示する git log
log 著者、コミッタを正規表現で検索してログを表示する git log --committer=regexp
log コミットメッセージを逆順にして表示する git log --reverse
log 指定した時間に当てはまるログを表示する git log --since=2011-2-6
log コミットログと変更の状態を表示する git log --stat
log コミットログを指定した数だけ表示する git log -n 5
log 特定のファイルのコミットログを表示する git log file.txt
merge 現在のブランチに指定したブランチへのコミットをマージする git merge ${ブランチ名}
mergetool コンフリクトをマージツールを利用して解決する
操作は、対話式になっており、指示にしたがうことでマージツールを実行する。
マージツールは実行時に、candicateに表示されているツールを利用することができる。
表示されるツールはインストールされていない場合もあるので、必要に応じてインストールする
コンフリクトを修正して、保存を行い、「git add xxx;git commit」することで、コンフリクト修正済みのファイルをコミットする		 git mergetool
git mergetool --tool=${ツール名}
mv ファイルを移動する git mv ${from} ${to}
pull ほかのリポジトリの変更点をローカルリポジトリにマージする git pull ${変更点の取り込み元リポジトリ}
push 指定のリポジトリに自分のリポジトリのデータを送信する。 git push ${送信先リポジトリ} ${送信元ブランチ}:${送信先ブランチ}
rebase rebaseを元に戻す git rebase --abort
rebase ブランチの派生元を変更 git rebase ${変更先ブランチ名}
reflog HEADの変遷を確認することができる（コミットではなく、HEADがどう変遷していったかを確認できる。
gcされる前であれば、必ず見つけることができる		 git reflog
reflog delete 不要なrefログを削除する git reflog delete ${削除するreflog(reflogでみれる2カラム目の値を指定する)}
reflog expire ログが保持される期間に関する操作を行う git reflog expire
remote 設定済みのリモートリポジトリのハンドル名を表示 git remote
remote 設定済みのリモートリポジトリのハンドル名とURLを表示 git remote -v
remote リモートリポジトリを追加 git remote add ${ハンドル名} ${URL}
remote リモートリポジトリを削除 git remote rm ${ハンドル名}
remote リモートリポジトリの詳細情報を表示 git remote show ${ハンドル名}
reset HEADの位置、インデックス、ワーキングツリーを3つ前の状態に戻す git reset --hard HEAD^3
reset HEADの位置、インデックスを2つ前に戻す。ワーキングツリーには影響は与えない git reset --mixed HEAD^^
reset HEADの位置を1つ前に戻す。インデックスとワーキングツリーには影響は与えない git reset --soft HEAD^
revert 特定のコミットを取り消す（ステージングには上がるがコミットは行わない） git revert -n ${コミットID}
revert 特定のコミットを取り消す（自動的にコミットを行う） git revert ${コミットID}
rm 指定したファイルを、ツリーから削除する git rm ${削除するファイル名}
show 最新のコミットのメッセージと変更点を表示する git show
show 指定したコミットのメッセージと変更点を表示する git show ${コミットID}
stash ステージされていない変更を一時的に保存する
行うごとに蓄積していく		 git stash
git stash save ${メッセージ}
stash clear 保存した変更をすべて捨てる git stash clear
stash drop 一番最近に保存した変更を捨てる git stash drop
stash list 保存したリストを確認する git stash list
stash pop 一番最近に保存した変更を元に戻す git stash pop
status 変更が加えられたファイルを表示する git status
status 変更が加えられたファイルを表示する（短縮形で表示） git status -s
tag タグ一覧を表示する git tag
tag HEAD にアノテーションつきのタグをつける git tag -a ${タグ名} -m "annotation"
tag タグを削除する git tag -d ${タグ名}
tag タグをパターンで検索して表示する git tag -l ${タグ名}
tag タグのリストとアノテーションを表示する git tag -n${表示行数} -l
tag HEAD にアノテーションなしのタグをつける git tag ${タグ名}
tag 特定のコミットにタグをつける git tag ${タグ名} some-commit
